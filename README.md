1. **기본 요구사항**
    - ✅  Todo 항목 추가, 목록 표시, 삭제, 완료 상태 표시 기능이 모두 구현되어 있나요?
    - ✅  UI(사용자 인터페이스) 및 UX(사용자 경험)이 명확한가요?
    - ✅  할일 입력, 삭제, 상태 변경 등의 기능이 정상 작동하나요?
2. **Typescript**
    - ✅  강의에서 배운 내용을 토대로 Typescript로 작성하셨나요?
    - ✅  Typescript의 타입 안정성을 활용하고 계신가요?(예를 들어 변수와 함수의 타입이 명시되어 있는지)
3. **레벨에 맞는 구현**
    - ✅  선택한 레벨(1~5)에 맞는 기능이 구현되어 있나요?
    - ✅  레벨에 따른 추가 요구사항(예: RTK, json-server, redux thunk, react-query 사용)이 충족되었나요?
4. **디자인 및 레이아웃 준수**
    - ✅  화면 구성이 요구 사항(최대 넓이 1200px, 최소 넓이 800px, 전체 화면 가운데 정렬)에 맞게 되어 있나요?
    - ✅  디자인이 사용자 친화적이며, 코드는 협업을 위하여 가독성과 접근성을 고려하여 구성되었나요?
5. **코드 구조 및 문서화**
    - ✅  컴포넌트 구조가 효율적으로 구성되었나요?
    - ❌  반복되는 부분이 적절히 분리되었나요?
    - ✅  프로젝트의 README 파일에 컴포넌트 구조 및 사용 방법이 명확하게 문서화되어 있나요?
  

 ### 어려웠던 점 
 level 1이 생각보다 빨리 끝나서 level 2로 넘어가고 싶었는데 리덕스 툴킷으로 바꾸려니 코드가 자꾸만 꼬여서 천천히 시도해보았습니다. 그런데 투두에 날짜 객체를 넣어둔 탓에 RTK의 리듀서 작동이 제대로 되지않았습니다.
 **'A non-serializable value was detected in the state, in the path: `todos.0.date`. Value: Mon Dec 18 2023 00:59:16 GMT+0900 (한국 표준시)'** 라는 에러가 발생하였고 찾아보니 **'Redux 상태는 순수하게 직렬화 가능한(serializable) 값만을 포함해야 하므로, 날짜 객체와 같은 비직렬화할 수 없는 값을 포함하는 것은 권장되지 않는다.'** 라는 것을 알게 되어 파일을 다 지우게 되었습니다. 브랜치명을 level-2로 올려서 차마 바꾸지는 못했는데 이 부분 양해 부탁드리겠습니다 ㅜㅜ
